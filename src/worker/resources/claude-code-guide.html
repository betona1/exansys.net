클로드코드란?

터미널에서 대화로 코딩하는 AI 개발 도구
💬

대화로 지시

"로그인 화면 만들어줘"처럼 한국어로 시키면 AI가 코드를 씁니다.

📂

내 폴더를 읽음

프로젝트 파일 전체를 이해하고 여러 파일을 한 번에 고칩니다.

⚡

직접 실행

파일 생성·수정, 명령 실행, 깃 커밋까지 스스로 처리합니다.

🧭
한 줄 요약 — 클로드코드 = 터미널 안에 사는 AI 개발 파트너. 창(터미널)을 하나 열고, 말로 시키면 됩니다.
2

설치하기

셋 중 하나만 — Windows 기준
추천 · 가장 쉬움

① PowerShell 한 줄

파워셸을 열고 아래를 붙여넣기. 자동으로 다 깔립니다.

PowerShell
PS> irm https://claude.ai/install.ps1 | iex
Windows 패키지

② winget

Windows 기본 패키지 관리자로 설치.

PowerShell
PS> winget install Anthropic.ClaudeCode
Node 사용자

③ npm

Node.js 22 이상 필요. 이미 Node가 있으면 이 방법.

Terminal
$ npm install -g @anthropic-ai/claude-code
🔑

로그인

설치 후 터미널에 claude 입력 → 브라우저가 열리면 로그인 한 번. 이후 자동 유지.

Terminal
$ claude
# 처음이면 자동으로 /login 안내 → 브라우저에서 로그인
🎫

필요한 것

사용하려면 유료 구독이 있어야 합니다.

Claude ProMaxTeam
⚠️
무료 요금제로는 클로드코드를 쓸 수 없습니다. Pro 이상 구독이 필요해요.

터미널에서 쓰기

명령창을 열고 말로 시키기
내 프로젝트 폴더 — Terminal
# 대화 시작 (그냥 실행)
$ claude

# 할 일을 바로 던지기
$ claude "버튼 색을 초록으로 바꿔줘"

# 이전 대화 이어서
$ claude -c
⌨️

대화 중 자주 쓰는 명령

채팅 안에서 / 를 치면 명령 목록이 뜹니다.

/login/clear/help

/clear = 대화 초기화 · /help = 도움말

🚪

기본 흐름

폴더로 이동

cd 내프로젝트

클로드 실행

claude 입력

말로 지시

원하는 걸 한국어로

💡
터미널이 무섭게 느껴져도 괜찮아요. 실제로 외울 명령은 cd, claude 둘뿐입니다. 나머지는 클로드에게 물어보세요.

VS Code에서 쓰기

코드 에디터 안에서 나란히 보며 작업

VS Code 설치

code.visualstudio.com 에서 내려받아 설치. (버전 1.98 이상 권장)

확장 탭 열기

왼쪽 블록 아이콘 클릭 또는 Ctrl+Shift+X

"Claude Code" 검색 → 설치

Anthropic 공식 확장을 설치합니다.

사이드 패널에서 대화

에디터 오른쪽에 클로드 패널이 열리고, 코드와 결과를 나란히 보며 작업합니다.

🖥️
터미널 방식과 같은 클로드코드입니다. 화면에서 코드 변화를 눈으로 보며 하고 싶으면 VS Code, 빠르게 치고 빠지려면 터미널.

GitHub 란?

코드의 클라우드 저장소 + 버전 기록
☁️

무엇인가

내 코드를 인터넷에 안전하게 보관하고, 언제 무엇을 바꿨는지 전부 기록하는 곳. 협업·백업·배포의 중심.

⏱️

왜 쓰나

실수해도 이전으로 되돌리고, 여러 기기·여러 사람이 같은 코드를 공유. EXANSYS 사이트도 GitHub에 올리면 자동 배포됩니다.

핵심 흐름 — 내 컴퓨터 ↔ GitHub

💻
내 컴퓨터코드 작성·수정
저장 기록→commit
📦
로컬 저장소변경 이력 쌓임
올리기→push
☁️
GitHub온라인 보관
🔁
반대로 GitHub의 최신 코드를 내 컴퓨터로 가져오는 건 pull. 커밋·푸시·풀 이 셋이 GitHub의 전부라고 봐도 됩니다.
🤝

클로드코드 + GitHub

복잡한 깃 명령도 클로드가 대신 처리합니다. 계정 연결은 gh(GitHub CLI)로 한 번만.

Terminal
$ gh auth login        # GitHub 계정 연결 (최초 1회)
$ claude "방금 바꾼 거 커밋하고 올려줘"  # 커밋·푸시 자동

WSL 이란? (Windows용 리눅스)

윈도우 안에서 리눅스를 돌리는 기능
🐧

무엇인가

Windows Subsystem for Linux — 윈도우를 끄지 않고 그 안에서 리눅스(우분투)를 쓰는 기능.

🛠️

왜 필요한가

많은 개발 도구가 리눅스 기준으로 만들어져 있어, WSL을 쓰면 명령어가 깔끔하게 맞아떨어지고 오류가 줄어듭니다.

설치 — 관리자 PowerShell에서

PowerShell을 관리자로 실행

시작 메뉴 → PowerShell 우클릭 → 관리자 권한으로 실행

한 줄 설치

아래 명령이 WSL + 우분투를 한 번에 깔아줍니다.

관리자 PowerShell
PS> wsl --install

재부팅 후 우분투 첫 설정

컴퓨터를 다시 켜면 우분투 창이 열립니다. 사용할 아이디·비밀번호를 정하면 끝.

WSL 안에서 클로드코드 설치

우분투 터미널에서 아래 한 줄.

Ubuntu (WSL)
$ curl -fsSL https://claude.ai/install.sh | bash
🧩
꼭 WSL이 필요한 건 아니에요. 윈도우에서 바로 써도 됩니다. 리눅스 기준 강의를 그대로 따라 하고 싶을 때 WSL을 쓰면 편합니다.
