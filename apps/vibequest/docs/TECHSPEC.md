## 1. 기존 용어 데이터 진단

기존 파일은 JSON 배열 367개로 파싱 가능하지만, 그대로 문제를 자동 생성하면 품질 문제가 발생한다.

### 1.1 발견된 구조 문제

1. `difficulty=2`가 334개로 전체의 약 91%를 차지한다. 난이도 적응 기능이 사실상 작동하기 어렵다.
2. `vibeCore=true`가 18개뿐이라 바이브코딩 핵심 용어 우선 학습이 충분하지 않다.
3. 서로 다른 개념을 한 카드에 합친 항목이 많다.
   - `MySQL / PostgreSQL`
   - `Firestore / 실시간 DB`
   - `React / React Native`
   - `SELECT / INSERT / UPDATE / DELETE`
   - `리텐션 / DAU·MAU`
   - `앱스토어 / 플레이스토어`
4. 동일 개념이 다른 카테고리에 반복된다.
   - Google Play Console
   - 백엔드, 프론트엔드, 풀스택
   - 빌드, 저장소, PR
5. 시점에 따라 바뀌는 정책을 고정 정의로 저장했다.
   - 개발자 계정 비용
   - Google Play 클로즈드 테스트 조건
   - 타겟 API 기준
6. `termKo`와 `termEn`의 방향이 일관되지 않다. 일부는 한국어, 일부는 발음, 일부는 설명이다.
7. 원본에서 `termEn`이 비어 있는 항목이 12개였다.
8. 정의가 너무 짧아 오답 선택지를 만들기 어려운 항목이 있다.
9. 모든 항목의 `source`가 사실상 같은 포괄 문구여서 개별 검증이 어렵다.
10. 퀴즈 생성에 필요한 별칭, 예시, 오개념, 변동성, 검수일 필드가 없다.

### 1.2 적용한 정리 원칙

- 한 항목은 한 개념만 설명한다.
- 동의어는 `aliases`에 저장한다.
- 비교가 필요한 개념은 별도 `comparePairId`로 연결한다.
- 정책·가격·버전처럼 바뀌는 내용은 `volatile=true`, `checkedAt`을 갖는다.
- 정의는 초보자가 읽을 수 있는 1~2문장으로 제한한다.
- 문제에 사용할 수 없는 항목은 `quizEnabled=false`로 제외할 수 있다.
- 신규 500개는 AI, 프롬프트, 에이전트, 프로그래밍, 웹, 백엔드, DB, DevOps, 보안, 모바일, UX·학습게임 분야에 균형 배치한다.

### 1.3 결과 파일

- `glossary_cleaned_367.json`: 기존 367개 교정본
- `glossary_additions_500.json`: 신규 500개
- `glossary_master_867.json`: 통합 867개
- `glossary_master_867.csv`: 엑셀·관리자 검수용
- `glossary_validation_report.json`: 자동 검증 결과

## 2. 제품 목표와 제외 범위

### 2.1 제품 목표

- 처음 보는 개발 용어를 두려움 없이 접하게 한다.
- 용어의 정의뿐 아니라 실제 상황에서 어떤 의미인지 구별하게 한다.
- 하루 3~7분 학습으로 반복 방문 습관을 만든다.
- 틀린 문제를 자동으로 다시 보여 장기 기억으로 전환한다.
- 사용자가 “내가 무엇을 모르는지” 확인할 수 있게 한다.
- 바이브코딩 도구에 지시할 때 필요한 최소 공용어를 익히게 한다.

### 2.2 비목표

- 실제 프로그래밍 언어 문법 전체를 가르치는 코딩 강의 앱이 아니다.
- 자격증 시험처럼 긴 문제와 점수 경쟁을 중심으로 하지 않는다.
- 사용자를 압박하는 하트 소진, 강제 광고, 유료 확률형 보상을 사용하지 않는다.
- AI가 검수 없이 새 문제를 즉시 운영 DB에 게시하지 않는다.

## 3. 대상 사용자

### P1. 완전 초보

- “API, DB, Git이 무엇인지”부터 모른다.
- 코드보다 용어가 먼저 장벽이다.
- 긴 설명보다 짧은 예시와 그림이 필요하다.

### P2. 바이브코딩 입문자

- Claude Code, Codex, Copilot, Cursor 같은 도구로 앱을 만들어 본다.
- 에러 메시지와 AI 답변 속 용어를 이해하고 싶다.
- 스펙, 컨텍스트, MCP, 테스트, 배포 용어가 특히 중요하다.

### P3. 비개발 직군

- 기획·디자인·운영 업무에서 개발자와 소통하려 한다.
- 실제 코드를 작성하지 않더라도 프론트엔드, 백엔드, API, DB 구조를 구분해야 한다.

## 4. UX 설계 원칙

### UX-01. 한 화면에는 한 가지 판단만 둔다

문제 화면에는 질문, 답변, 진행 상태만 표시한다. 설명, 광고, 이벤트 배너를 동시에 노출하지 않는다.

### UX-02. 5초 안에 답할 수 있는 문제부터 시작한다

초기 세션은 O/X, 매칭, 스와이프를 중심으로 구성한다. 주관식과 퍼즐은 숙련 후 점진적으로 해금한다.

### UX-03. 틀림은 실패가 아니라 다음 복습 신호다

오답 시 빨간 화면을 오래 보여주지 않는다. “거의 맞았어요” 같은 모호한 칭찬 대신 정확한 이유와 비교 개념을 보여준다.

### UX-04. 보상 애니메이션은 700ms 이내이며 진행을 막지 않는다

보석은 정답 버튼 위치에서 상단 보유량으로 날아간다. 사용자는 애니메이션 중에도 다음 버튼을 누를 수 있다.

### UX-05. 색상만으로 정답을 구분하지 않는다

정답은 체크 아이콘과 “정답” 텍스트, 오답은 X 아이콘과 이유를 함께 표시한다.

### UX-06. 모션·소리·진동은 각각 끌 수 있다

시스템의 모션 감소 설정을 존중하고, 학습 정보는 소리 없이도 이해 가능해야 한다.

### UX-07. 모르는 것을 선택해도 손해가 없다

`알아요 / 헷갈려요 / 처음 봐요` 진단은 점수에 반영하지 않는다. 솔직한 선택이 더 좋은 추천으로 이어져야 한다.

## 5. 핵심 학습 루프

```text
앱 실행
  → 오늘의 3분 미션 제안
  → 5~8문제 풀이
  → 문제별 즉시 피드백 + 보석/XP
  → 틀리거나 자신감 낮은 용어 자동 저장
  → 세션 종료 요약
  → 다음 복습 시간 예약
  → 용어 도감 해금 및 퀘스트 진행
```

### 5.1 세션 기본값

- 기본 문제 수: 7개
- 목표 시간: 3분
- 최대 연속 설명 길이: 2문장
- 새 용어:복습 용어 비율
  - 신규 사용자: 60:40
  - 7일 이후: 40:60
- 같은 용어는 한 세션에서 최대 2회 노출한다.
- 오답 직후 같은 문제를 그대로 반복하지 않고 다른 형식으로 2~4문제 뒤에 다시 제시한다.

### 5.2 세션 종류

| 세션 | 구성 | 목적 |
|---|---|---|
| 오늘의 미션 | 추천 7문제 | 기본 습관 |
| 빠른 복습 | 만기 복습 5문제 | 기억 유지 |
| 새 용어 탐험 | 새 용어 8개 | 범위 확장 |
| 오답 수리 | 최근 오답 5개 | 오개념 수정 |
| 보스 스테이지 | 혼합 12문제 | 단원 숙련 확인 |
| 자유 연습 | 카테고리·난이도 선택 | 자율 학습 |

## 7. 온보딩과 언노운 진단

### 7.1 첫 실행 흐름

1. 로고 및 한 문장 가치 제안
2. “하루 몇 분?” 선택: 3분 / 5분 / 10분
3. 관심 분야 최대 3개 선택
   - AI·에이전트
   - 앱 만들기
   - 웹·API
   - 데이터베이스
   - Git·배포
   - UX·UI
4. **언노운 진단 12장**
5. 첫 3분 미션 즉시 시작
6. 미션 종료 후 게스트 유지 또는 계정 연결 제안

### 7.2 언노운 진단 UI

용어 카드 하나를 보여주고 아래 3개 버튼을 제공한다.

- 알아요
- 헷갈려요
- 처음 봐요

#### 동작

- `알아요`: 1개의 확인 문제를 나중에 출제한다.
- `헷갈려요`: 24시간 안에 정의·비교 문제를 출제한다.
- `처음 봐요`: 즉시 15초 학습 카드 후 쉬운 인식 문제를 출제한다.
- 진단 결과는 점수나 스트릭에 영향을 주지 않는다.

### 7.3 언노운 선택 알고리즘

초기 12개는 다음 비율로 샘플링한다.

- `vibeCore=true`: 6개
- 사용자가 선택한 관심 분야: 4개
- 공통 개발 기초: 2개

서로 비슷한 용어를 한 번에 두 개 이상 내지 않는다. 예: 인증과 인가를 첫 진단 카드에 연속 배치하지 않는다.

## 8. 문제 유형 명세

## 8.1 O/X 퀴즈

### 목적

정의의 핵심 사실을 빠르게 인식하게 한다.

### UI

- 상단: 진행 `3 / 7`, 콤보, 종료 버튼
- 중앙: 문장 카드
- 하단: 큰 O, X 버튼
- 버튼 높이 최소 56dp

### 규칙

- 부정문을 연속으로 2개 이상 내지 않는다.
- “항상”, “절대” 같은 함정 표현을 남용하지 않는다.
- 오답 시 잘못된 부분을 굵게 표시한 교정 문장을 보여준다.

### 예

> “MCP는 AI가 외부 도구와 데이터를 표준 방식으로 연결하도록 돕는다.”

정답: O

---

## 8.2 4지선다

### 목적

비슷한 개념을 구분하고 정확한 정의를 선택하게 한다.

### UI

- 선택지는 세로 4개 카드
- 한 줄에 한 선택지
- 선택 후 즉시 정답 표시
- 긴 선택지는 최대 3줄, 전체 문구가 보여야 한다.

### 오답 생성 규칙

- 같은 카테고리·비슷한 난이도에서 선택한다.
- 완전히 엉뚱한 오답은 1개 이하로 제한한다.
- 자주 혼동하는 개념을 `confusionSet`으로 관리한다.
- 정답 길이만 유독 길거나 짧지 않게 한다.

### 변형

- 용어 → 정의 선택
- 정의 → 용어 선택
- 상황 → 필요한 개념 선택
- 잘못된 설명 하나 찾기

## 8.3 주관식

### 목적

단순 인식이 아니라 기억에서 용어를 직접 인출하게 한다.

### 기본 입력 방식

- 완전 자유 입력보다 **첫 글자 칩 + 자동완성 후보**를 기본으로 한다.
- 입력 2글자 이후 최대 5개 후보를 보여준다.
- 한글·영문·약어·허용 별칭을 모두 정답 처리한다.

### 채점

1. 정확 일치
2. 대소문자·공백·하이픈 무시
3. `aliases` 일치
4. 오타 거리 허용
   - 4자 이하: 0자
   - 5~8자: 1자
   - 9자 이상: 2자
5. 의미가 다른 유사 단어는 자동 정답 처리하지 않는다.

### 힌트 사다리

1. 첫 글자
2. 글자 수
3. 카테고리
4. 정의의 핵심 단어
5. 보기 4개로 전환

힌트 사용은 학습을 막지 않으며 XP만 소폭 조정한다.

## 8.4 코드그리드: 십자말 대체 설계

기존 전통 십자말은 작은 모바일 화면에서 칸 선택, 키보드, 가로·세로 전환이 어렵다. 기본 모드에서는 다음 방식으로 바꾼다.

### 단계 1. 워드 스트립

- 빈칸이 있는 가로 단어 한 개만 표시
- 아래 글자 칩을 순서대로 탭
- 정답 글자는 즉시 고정
- 3~6글자 용어 중심

### 단계 2. 미니 교차

- 2~3개 용어만 교차
- 현재 풀 용어를 굵은 테두리로 강조
- 다른 칸은 흐리게 표시
- 교차 글자는 미리 제공해 단서로 사용

### 단계 3. 코드그리드

- 최대 5개 용어
- 전체 격자는 보이지만 한 번에 한 단서만 활성화
- 키보드 직접 입력 대신 글자칩·드래그를 기본으로 제공
- 정답 단어는 자동 완성되고 다음 단서로 이동

### 접근성 대체 보기

`격자 보기 / 목록 보기` 전환을 제공한다. 목록 보기에서는 각 단어를 독립된 빈칸 문제로 풀 수 있어 스크린 리더와 큰 글자에서도 동등하게 완료할 수 있다.

### 금지 사항

- 10개 이상 단어가 얽힌 대형 격자
- 작은 칸을 직접 정확히 탭해야만 하는 조작
- 정답 입력 후 수동으로 다음 칸 이동
- 가로/세로 전환 아이콘만 제공하고 텍스트 설명 없음
- 퍼즐 미완료 시 다른 콘텐츠 잠금

## 8.5 추가 권장 게임 모드

### A. 용어 스와이프

카드를 좌우로 넘기는 방식이 아니라 하단 3버튼을 기본으로 사용한다.

- 알아요
- 헷갈려요
- 처음 봐요

스와이프 제스처는 보조 기능으로만 제공한다. 발견형 진단과 빠른 분류에 적합하다.

### B. 버그 헌트

짧은 코드, 에러 메시지, UI 설명 속 잘못된 용어를 탭한다.

예: `401은 권한은 있지만 접근할 수 없다는 뜻이다.`에서 잘못된 설명을 찾는다.

### C. 프롬프트 조립소

역할, 목표, 컨텍스트, 제약, 출력 형식 블록을 올바른 순서로 배치한다.

학습 목표: 좋은 프롬프트는 긴 문장이 아니라 필요한 정보 구조를 갖는다는 점을 익힌다.

### D. 컨텍스트 가방 싸기

“로그인 오류를 고쳐줘” 같은 작업에 필요한 파일과 정보를 선택한다.

- 로그인 화면 코드
- 인증 API 응답
- 에러 로그
- 무관한 이미지 파일

정답은 하나가 아니라 필수·도움·불필요로 분류할 수 있다.

### E. 에이전트 플로

도구 호출 순서를 카드로 배치한다.

`파일 읽기 → 원인 찾기 → 수정 → 테스트 → 결과 요약`

### F. 짝 맞추기

용어와 정의, 약어와 풀네임, 오류 코드와 의미를 매칭한다. 카드 수는 6~10장으로 제한한다.

### G. 카테고리 분류

용어 칩을 AI, DB, Git, UX 같은 영역에 드래그한다. 드래그가 어려운 사용자는 탭 후 카테고리 선택을 사용할 수 있다.

### H. 한 줄 설명

“API를 초등학생에게 설명한다면?”처럼 자신의 말로 한 줄을 입력한다. MVP에서는 AI 자동 채점 대신 핵심 단어 포함 여부와 자기평가를 사용한다. 이후 검수된 AI 피드백을 선택 기능으로 추가한다.

### I. 보스 스테이지

12문제 혼합형이다.

- 1~3: 인식
- 4~6: 구분
- 7~9: 상황 적용
- 10~11: 주관식
- 12: 프롬프트·에이전트 플로

하트 대신 방패 3칸을 시각 효과로 사용하되, 방패가 모두 깨져도 학습은 계속된다. 최종 보너스만 달라진다.

## 9. 적응형 학습 엔진

### 9.1 용어별 상태

```text
NEW         처음 접함
LEARNING    같은 세션 또는 당일 학습 중
REVIEWING   간격 반복 대상
MASTERED    여러 간격에서 안정적으로 정답
LAPSED      숙련 후 다시 틀림
```

### 9.2 숙련도 점수

각 용어는 `masteryScore` 0~100을 가진다.

```text
기본 정답 점수
- O/X 또는 매칭 정답: +5
- 4지선다 정답: +7
- 주관식 무힌트 정답: +12
- 상황형 정답: +10

보정
- 3초 미만 무작위 추정 가능성: +0~+3 제한
- 자신감 높음 + 정답: +2
- 자신감 높음 + 오답: -8 및 오개념 표시
- 힌트 사용: 획득 점수 30~70% 적용
- 오답: -5, 단 0 미만으로 내려가지 않음
```

### 9.3 MVP 복습 간격

| 결과 | 다음 복습 |
|---|---|
| 처음 봄 | 같은 세션 2~4문제 뒤 |
| 오답 | 10분 이내 또는 다음 세션 첫 부분 |
| 정답·낮은 자신감 | 1일 후 |
| 정답·보통 자신감 | 3일 후 |
| 연속 2회 정답 | 7일 후 |
| 연속 3회 이상 | 21일 후 |
| MASTERED 후 오답 | 1일 후, 상태 LAPSED |

### 9.4 문제 형식 승급

```text
NEW → 학습 카드 + O/X
LEARNING → 4지선다 + 매칭
REVIEWING → 정의 역방향 + 주관식
MASTERED → 상황형 + 비교형 + 교차 학습
```

### 9.5 난이도 적응

최근 10문제 기준으로 다음을 적용한다.

- 정답률 90% 이상, 평균 응답 5초 이하: 한 단계 상승
- 정답률 60~89%: 유지
- 정답률 60% 미만: 새 용어 중단, 복습·힌트 강화
- 같은 카테고리 3연속 오답: 20초 미니 설명 카드 삽입
- 자신감 높음 오답: 해당 혼동쌍 문제 우선 배치

## 10. 보상과 게임 경제

### 10.1 보상 종류

| 보상 | 획득 | 사용 |
|---|---|---|
| XP | 문제 풀이, 퀘스트 | 레벨 상승 |
| 보석 | 정답, 세션 완료 | 테마·프로필 장식·힌트 |
| 지식 조각 | 특정 카테고리 숙련 | 용어 도감 카드 진화 |
| 배지 | 업적 | 프로필 전시 |
| 스트릭 보호권 | 주간 목표 | 연속 학습 1회 보호 |

### 10.2 보석 연출

#### 정답 1개

- 3~5개의 작은 보석 입자
- 선택지 위치에서 상단 보석 카운터로 곡선 이동
- 450~650ms
- 약한 성공 햅틱 1회
- 카운터가 `+2` 형태로 잠깐 증가

#### 콤보 3 이상

- 보석 8~12개
- 화면 가장자리까지 퍼지지 않음
- `COMBO x3` 텍스트 500ms
- 소리는 음소거 가능

#### 세션 완료

- 획득 보석과 XP를 숫자로 먼저 표시
- 상자 열기보다 “오늘 배운 용어 카드”를 앞에 보여준다.
- 애니메이션 건너뛰기 버튼 제공

### 10.3 콤보 규칙

- 정답 시 +1
- 오답 시 0으로 초기화하지 않고 1단계 감소
- 힌트 사용 정답은 콤보 유지, 증가 없음
- 시간 압박은 기본 모드에서 사용하지 않는다.

### 10.4 학습을 해치지 않는 원칙

- 보석을 얻기 위해 이미 아는 쉬운 문제만 반복할 수 없게 보상 감쇠 적용
- 복습 만기 문제에는 1.2배 보상
- 오답 수리 완료에는 추가 보상
- 광고 시청으로 정답 처리 금지
- 결제로 숙련도나 정답률을 사는 기능 금지

## 11.1 홈 화면

### 구성

1. 상단 인사와 레벨

- `FR-HOME-003`: 복습 만기 항목이 있으면 미션 구성에 우선 포함한다.
- `FR-HOME-004`: 네트워크가 없어도 학습 시작이 가능해야 한다.
- `FR-HOME-005`: 스트릭 문구는 협박형 표현을 사용하지 않는다.

## 11.2 학습 모드 화면

- 추천 모드 상단 배치
- 잠금 모드는 잠금 이유와 해금 조건 표시
- 카테고리 검색 및 필터
- 예상 시간과 문제 수 표시

`FR-LEARN-001`: 모드 카드는 최대 6개만 먼저 보여주고 나머지는 더보기로 둔다.  
`FR-LEARN-002`: 마지막으로 사용한 설정을 기억한다.  
`FR-LEARN-003`: 자유 연습 결과도 숙련도에 반영하되 반복 보상은 감쇠한다.

## 11.3 문제 화면

- 상단 왼쪽 닫기
- 중앙 진행바
- 오른쪽 콤보
- 질문 카드
- 답변 영역
- 하단 피드백 시트

`FR-QUIZ-001`: 답변 전 다음 버튼을 표시하지 않는다.  
`FR-QUIZ-002`: 답변 후 정답·설명·다음 버튼을 한 영역에서 표시한다.  
`FR-QUIZ-003`: 앱이 백그라운드로 갔다 돌아와도 현재 문제를 복원한다.  
`FR-QUIZ-004`: 중복 탭으로 답안이 두 번 제출되지 않아야 한다.  
`FR-QUIZ-005`: 정답 제출 후 100ms 이내 시각적 눌림 피드백을 준다.  
`FR-QUIZ-006`: 보상 애니메이션이 다음 문제 이동을 막지 않는다.

## 12. UI 디자인 시스템

### 12.1 시각 방향

- 밝고 선명한 한 가지 주조색
- 넓은 여백과 큰 둥근 카드
- 모양과 크기의 대비를 활용한 표현적 UI
- 장식보다 질문 가독성 우선
- 그라데이션은 홈·보상 화면에만 제한적으로 사용

### 12.2 타이포그래피 권장

| 용도 | 크기 |
|---|---|
| 문제 문장 | 22~26sp |
| 선택지 | 17~19sp |
| 본문 설명 | 16~18sp |
| 보조 정보 | 최소 14sp |
| 숫자 보상 | 24~32sp |

시스템 글자 크기 200%에서도 주요 기능을 사용할 수 있어야 한다.

### 12.3 터치와 간격

- 모든 핵심 터치 영역 최소 48×48dp
- 선택지 카드 높이 최소 56dp
- 선택지 간격 10~12dp
- 화면 좌우 기본 여백 20dp
- 하단 시스템 영역과 겹치지 않게 안전 영역 적용

### 12.4 상태 색상

- 정답: 색 + 체크 + 텍스트
- 오답: 색 + X + 교정 설명
- 선택 전: 중립 배경
- 비활성: 대비를 유지하되 투명도만 과도하게 낮추지 않는다.

### 12.5 모션

- 일반 전환: 200~300ms
- 정답 카드 반응: 150~250ms
- 보석 이동: 450~650ms
- 세션 완료: 최대 1.2초, 건너뛰기 가능
- 흔들림 애니메이션은 오답 시 1회, 4px 이하

### 12.6 햅틱

- 선택: 매우 약한 탭
- 정답: 짧은 성공 피드백
- 오답: 강한 경고 대신 짧은 중립 피드백
- 보스 완료: 패턴 1회

햅틱만으로 정보를 전달하지 않는다.

## 14. 콘텐츠 생성·검수 파이프라인

### 14.1 자동 생성 가능 범위

AI가 초안을 만들 수 있는 항목:

- O/X 문장
- 4지선다 오답 후보
- 쉬운 예시
- 비교 문제
- 상황형 문제 초안

### 14.2 반드시 사람 검수가 필요한 항목

- 정답 자체
- 정책·가격·버전 정보
- 보안 및 개인정보 용어
- 서로 비슷한 개념의 차이
- 주관식 허용 별칭
- 오답 선택지가 실제로 오개념을 유도하는지 여부

### 14.3 문제 검수 상태

```text
DRAFT → AUTO_CHECKED → HUMAN_REVIEWED → APPROVED → RETIRED
```

### 14.4 자동 검사

- 정답이 선택지에 정확히 1개 존재하는가
- 선택지 중복이 없는가
- 용어와 정의가 반대로 연결되지 않았는가
- 부정문이 이중 부정인가
- 정의에 정답 단어가 그대로 노출되는가
- 변동성 항목의 검수일이 만료되었는가
- 난이도와 문제 유형이 맞는가
- 특정 선택지만 지나치게 긴가

### 14.5 변동성 콘텐츠

`volatile=true`인 항목은 앱 콘텐츠 업데이트 시 우선 점검한다.

- 스토어 정책: 30일
- 요금·가격: 30일
- API 버전·제출 기준: 60일
- 일반 기술 개념: 365일

## 22. 수용 기준

### AC-001 첫 학습

- 새 사용자는 앱 설치 후 3번 이하의 탭으로 첫 문제를 시작할 수 있다.
- 회원가입 없이 진행 가능하다.

### AC-002 언노운 진단

- 12개 카드를 90초 이내 완료할 수 있다.
- `처음 봐요`를 선택해도 XP나 스트릭 손실이 없다.
- 결과가 첫 미션 문제 구성에 반영된다.

### AC-003 문제 피드백

- 답 제출 후 300ms 이내 정답 상태가 표시된다.
- 모든 오답에는 이유 또는 올바른 설명이 있다.
- 애니메이션 중 다음 문제로 이동할 수 있다.

### AC-004 주관식

- 한글, 영문, 허용 약어가 모두 채점된다.
- 오타 허용 규칙이 테스트로 검증된다.
- 유사하지만 다른 용어를 정답 처리하지 않는다.

### AC-005 코드그리드

- 한 번에 한 단서만 활성화된다.
- 목록 보기로 동일한 퍼즐을 완료할 수 있다.
- 200% 글자 크기에서도 가로 스크롤 없이 핵심 조작이 가능하다.

### AC-006 복습

- 오답 용어가 다음 세션 또는 정해진 간격에 다시 등장한다.
- 앱 종료 후에도 복습 일정이 유지된다.
- 같은 문제 문장만 반복하지 않고 다른 유형으로 재출제된다.

### AC-007 오프라인

- 네트워크가 없는 상태에서 미션 시작·완료·보상 저장이 가능하다.
- 연결 복구 후 중복 보상 없이 동기화된다.

## 23. 테스트 시나리오

### TS-001 빠른 연속 탭

사용자가 같은 선택지를 빠르게 5회 눌러도 답안은 1회만 기록되어야 한다.

### TS-002 앱 중단 복원

문제 4/7에서 앱이 백그라운드 종료된 뒤 재실행하면 해당 세션 복원 또는 안전한 재시작 선택을 제공해야 한다.

### TS-003 콘텐츠 업데이트 실패

새 용어 JSON의 해시가 다르면 적용하지 않고 기존 콘텐츠로 정상 실행되어야 한다.

### TS-004 큰 글자

시스템 글자 200%에서 4지선다 선택지 전체 문구가 읽히고 버튼이 겹치지 않아야 한다.

### TS-005 스크린 리더 코드그리드

격자 대신 목록 보기로 단서·빈칸·제출 버튼을 순서대로 탐색할 수 있어야 한다.

### TS-006 숙련 후 오답

MASTERED 용어를 틀리면 LAPSED로 바뀌고 다음 복습이 1일 이내 예약되어야 한다.

### TS-007 정책 용어 만료

`volatile=true`이고 검수 기간이 지난 용어는 관리자 검수 목록에 자동 표시되어야 한다.
