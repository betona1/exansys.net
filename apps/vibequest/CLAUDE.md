전체 기능 명세는 `TECHSPEC.md`(확정판)과 `docs/TECHSPEC.md`(원문 v1.0)를 따른다.

## 2. 확정 결정 (변경 시 반드시 사용자 승인)

| 분석 | MVP 최소화. Firebase 도입은 사용자 승인 후 |

## 3. 기술 스택

- Flutter (Dart) · **Riverpod**(상태) · **go_router**(내비) · **Drift+SQLite**(로컬 DB)
- 용어 데이터: 번들 JSON (`data/glossary_v2.*.json` → 앱 assets)
- 보석 파티클 등 이펙트는 코드 기반(이미지 에셋 최소화)
- 기존 `apps/techdex/flutter_app`의 사운드(WAV 합성)·아이콘 제작 노하우는 재활용 가능하되, **앱은 새로 시작**

## 4. 콘텐츠 데이터 규칙

- 마스터: `data/glossary_v2.base.json` (867개, 11개 표준 도메인, 트랙 A~E 태깅)
- 보강본: `data/glossary_v2.enriched.json` — whyItMatters·example·confusionSet·aliases·난이도 재평가
- **AI 초안 → 사람 검수** 원칙 (스펙 §14): 정답·정책정보·혼동쌍은 반드시 검수 후 배포
- `volatile=true` 항목은 정의에 고정 사실 저장 금지, `checkedAt` 관리
- 한 항목 = 한 개념. 동의어는 `aliases`, 비교는 `confusionSet`
- 사용자 숙련도 저장 키는 `id` (termId) — 용어 삭제 시 즉시 제거 대신 `retired=true`

## 5. UX 불변 원칙 (스펙 §4·§12·§18 요약)

- 한 화면 한 판단. 문제 화면에 광고·배너 금지
- 오답은 처벌이 아니라 복습 신호 — 실패 문구 금지, 이유·비교 개념 표시
- 보상 애니메이션 700ms 이내, 진행을 막지 않음
- 색만으로 정답 구분 금지 (아이콘+텍스트 병행)
- 터치 48dp+, 선택지 56dp+, 글자 200% 대응, 모션·소리·햅틱 각각 끄기 가능
- 스트릭 문구는 협박형 금지 ("하루 쉬어도 괜찮아요")

## 6. 구현 우선순위 (스펙 §27 — 이 순서를 지킨다)

1. 콘텐츠 스키마 + 867 import (Drift)
2. 로컬 DB·숙련도 상태 (NEW→LEARNING→REVIEWING→MASTERED→LAPSED)
3. 홈 + 오늘의 미션 생성기
4. O/X + 4지선다
5. 주관식(별칭·오타 채점)
6. 즉시 피드백 + 보석 파티클
7. 오답 수리 + 복습 스케줄
8. 용어 도감
9. 온보딩 언노운 진단
10. 워드 스트립
11. 접근성·모션감소·오프라인 검증
12. 분석 이벤트 + 콘텐츠 업데이트
13. 미니 교차 코드그리드
14. 프롬프트 조립소·컨텍스트 가방

## 7. 하지 말 것

- 하트·생명·학습 차단 페널티 구현 금지
- AI 생성 문제를 검수 없이 운영 데이터에 게시 금지
- 광고 시청으로 정답 처리 금지, 결제로 숙련도 구매 금지
- 10개 이상 단어가 얽힌 대형 십자 격자 금지
- 정답률을 전면에 크게 표시하지 않기 (학습량·복습 우선)
- 세션 중 네트워크 의존 금지 (오프라인 필수)
