### 개발/에이전트 경험
- AX(에이전트 경험) | AX | 고급 | - | 환경이 에이전트가 잘 일하도록 갖춰진 정도 — 점검·아키텍처·여유 컨텍스트.
- DX(개발자 경험) | DX | 중급 | - | 코드베이스와 도구가 사람이 잘 일하게 해주는 정도 — 문서·피드백 속도·오류 메시지.

### 권한·안전
- 권한 모드 | Permission mode | 중급 | - | 에이전트 모드 중 권한 부분 — 어떤 도구 호출이 승인 요청을 띄우고 어떤 건 자동 실행되는지.
- 권한 요청 | Permission request | 중급 | - | 사전 승인되지 않은 도구 호출 전에 하네스가 사용자에게 보이는 확인. 사람을 개입시키는 장치.
- 샌드박스 | Sandbox | 중급 | 필수 | 에이전트가 갇혀 도는 격리 환경(컨테이너·VM·제한 셸). 행동의 피해 범위를 제한한다.
- 에이전트 모드 | Agent mode | 중급 | 필수 | 권한 모드와 시스템 프롬프트에 넣는 행동 지침을 묶은 프리셋. 세션 중 전환할 수 있다.

### 비용·토큰경제
- 입력 토큰 | Input tokens | 중급 | 필수 | 매 요청에서 하네스가 보내는 토큰. 출력 토큰보다 싸게 과금된다.
- 추론량(노력) | Effort | 중급 | 필수 | 답하기 전 모델이 얼마나 생각할지 조절하는 다이얼. 더 쓰면 출력 토큰을 더 써서 어려운 문제에 유리.
- 출력 토큰 | Output tokens | 중급 | 필수 | 모델이 생성해 돌려주는 토큰. 연산이 더 들어 입력보다 비싸게 과금된다.
- 캐시 토큰 | Cache tokens | 고급 | - | 이전 요청에서 제공자가 캐시해둔 입력 토큰. 훨씬 낮은 요율로 과금된다.
- 프리픽스 캐시 | Prefix cache | 고급 | - | 공유하는 앞부분을 다시 처리하지 않게 해주는 제공자 측 저장소. 그 토큰을 싸게 과금한다.

### 세션·핸드오프
- 무상태 | Stateless | 중급 | - | 정보를 이어가지 않음. 모델은 요청 간 무상태, 에이전트는 기본적으로 세션 간 무상태.
- 상태 유지 | Stateful | 중급 | - | 정보를 앞으로 이어감. 세션은 턴 사이에 상태를 유지한다.
- 세션 | Session | 초급 | 필수 | 에이전트와의 한 번의 작업 구간. 비어서 시작해 쌓이다가 초기화·종료·압축으로 끝난다.
- 스펙 | Spec | 고급 | - | 여러 세션짜리 작업을 설명한 핸드오프 문서. '무엇을' 만드는지를 담고 티켓들로 구성된다.
- 턴 | Turn | 중급 | - | 사용자 메시지 하나와 그에 대한 에이전트 응답 전부. 안에 여러 번의 요청이 있을 수 있다.
- 티켓 | Ticket | 고급 | - | 한 세션 분량의 작업을 정한 핸드오프 문서. 스펙에 붙거나 홀로 서며 서로 막을 수 있다.
- 핸드오프 | Handoff | 고급 | - | 한 세션의 컨텍스트를 다른 세션으로 넘기는 것. 돌아올 길은 없다.
- 핸드오프 문서 | Handoff artifact | 고급 | - | 한 세션이 써서 다른 세션이 읽게 하는, 인수인계용 문서.

### 작업 방식
- AFK(자리 비움) | AFK | 중급 | 필수 | 세션을 시작해두고 자리를 떠 에이전트를 무인으로 돌리는 방식. 여러 개를 병렬로 돌리는 처리량의 비결.
- 그릴링(캐묻기) | Grilling | 고급 | - | 디자인 컨셉을 잡는 기법. 에이전트가 한 번에 하나씩 소크라테스식으로 사용자를 인터뷰한다.
- 디자인 컨셉 | Design concept | 고급 | - | 무엇을 만들지에 대한, 사용자와 에이전트가 공유하는 이해. 산출물과는 별개다.
- 바이브코딩 | Vibe coding | 초급 | 필수 | 에이전트의 코드를 사람 검토 없이 받아들이는 방식. diff를 불투명하게 취급한다.
- 사람 개입(HITL) | Human-in-the-loop | 중급 | - | 세션 중 사람이 실시간으로 검토·수정·협업하며 에이전트와 짝을 이루는 방식.
- 사람 검토 | Human review | 중급 | 필수 | 에이전트가 만든 코드를 사람이 직접 읽고 판단하는 것. diff를 읽어야 인정, 요약만 읽으면 아니다.
- 자동 리뷰 | Automated review | 고급 | - | 다른 모델·프롬프트의 에이전트가 에이전트 결과를 검토하는 것. 판단이 들어가 비결정적이다.
- 자동 점검 | Automated check | 중급 | - | 환경에서 돌아가는 결정론적 검증 — 테스트·타입체크·린트·빌드. 합불만 있고 판단은 없다.
- 프로토타이핑 | Prototyping | 중급 | - | 말로는 부족할 때 에이전트에게 거칠고 빠른 버전을 만들게 해, 이야기할 실물을 얻는 것.

### 컨텍스트·메모리
- 1차 자료 | Primary source | 고급 | - | 코드·원본 데이터 그 자체. 완전하고 권위 있지만 컨텍스트에 넣기엔 비싸다.
- 2차 자료 | Secondary source | 고급 | - | 1차 자료를 요약·설명한 것. 싸게 넣지만 구조상 손실이 있다.
- 맥락 지식 | Contextual knowledge | 고급 | - | 지금 컨텍스트에서 바로 읽을 수 있는 사실. 파라미터 지식의 반대편.
- 메모리 시스템 | Memory system | 고급 | - | 세션이 끝나도 정보를 환경에 저장·재로딩해 에이전트를 이어지게 하려는 장치.
- 스마트존·멍청존 | Smart zone | 고급 | - | 세션 초반엔 또렷하지만 길어지면 흐려져 실수가 늘어나는 구간.
- 자동 컴팩션 | Autocompact | 고급 | - | 컨텍스트가 거의 찼을 때 하네스가 자동으로 실행하는 압축.
- 점진적 공개 | Progressive disclosure | 중급 | - | 지금 필요한 컨텍스트만 넣고 나머지는 포인터로 남겨, 필요할 때 불러오는 방식.
- 주의 관계 | Attention relationship | 중급 | - | 두 토큰 사이의 짝. 관련 있는 짝이 서로 더 강하게 영향을 준다. N개 토큰이면 약 N²개.
- 주의력 예산 | Attention budget | 고급 | - | 토큰 하나가 나머지에 미칠 수 있는 영향력의 총량. 토큰당 정해져 있어 컨텍스트가 커져도 늘지 않는다.
- 주의력 저하 | Attention degradation | 고급 | - | 세션이 길어질수록 각 토큰의 주의력이 더 많은 경쟁자에 분산돼 의미 있는 연결의 신호가 약해진다.
- 지식 컷오프 | Knowledge cutoff | 중급 | - | 그 이후의 지식은 모델이 학습하지 못한 기준 날짜. 이후의 라이브러리·API는 지어낼 위험이 있다.
- 컨텍스트 윈도우 | Context window | 중급 | 필수 | 모델이 한 요청에서 보는 전부. 유한하고 모델마다 다르며, 모델이 세상을 인식하는 유일한 창.
- 컨텍스트 포인터 | Context pointer | 고급 | - | 한 문서가 다른 문서를 가리키는 언급. 필요할 때만 그 내용을 컨텍스트로 불러오게 한다.
- 컴팩션(압축) | Compaction | 고급 | 필수 | 이전 대화를 요약해 새 세션의 출발점으로 삼는 것. 여유는 늘지만 세부는 손실된다.
- 클리어링(초기화) | Clearing | 중급 | - | 현재 세션을 끝내고 빈 컨텍스트로 새로 시작하는 것.
- 파라미터 지식 | Parametric knowledge | 고급 | - | 학습으로 파라미터에 저장된 지식. 학습 시점에 고정된다. 맥락 지식의 반대편.

### 품질·리스크
- 아첨(굽신) | Sycophancy | 중급 | - | 자신 있게 동조하는 출력. 사람이 좋아한 답(동의 포함)을 선호하도록 학습된 탓이다.

### 하네스·도구
- AGENTS.md | AGENTS.md | 중급 | - | 세션 시작 때 하네스가 컨텍스트에 넣는 프로젝트 상시 안내 파일.
- 도구 | Tool | 초급 | 필수 | 에이전트가 호출하도록 하네스가 노출한 기능(읽기·쓰기·실행·검색). 환경을 인식·조작하는 수단.
- 도구 결과 | Tool result | 중급 | - | 도구 실행 후 하네스가 돌려주는 것(파일 내용·출력·오류). 에이전트가 환경을 보는 유일한 창.
- 도구 호출 | Tool call | 중급 | - | 모델이 도구 이름과 인자를 출력한 것. 사실은 구조화된 텍스트일 뿐, 하네스가 읽어 실행한다.
- 모델 요청 | Model provider request | 중급 | - | 하네스와 제공자 간 한 번의 왕복. 컨텍스트를 보내면 응답 하나가 온다.
- 모델 제공자 | Model provider | 중급 | - | 추론용 모델을 서빙하는 곳. 보통 원격(Anthropic·OpenAI·Google), 로컬(Ollama)도 가능.
- 서브에이전트 | Subagent | 고급 | - | 다른 에이전트가 도구 호출로 띄운 에이전트. 자기 세션에서 돌고 결과 하나만 보고하며, 더 하위는 못 띄운다.
- 스킬 | Skill | 고급 | 필수 | 하나의 단위로 묶은 가르칠 수 있는 능력. 포인터로 불러오기 전엔 컨텍스트 밖에 둔다.
- 시스템 프롬프트 | System prompt | 중급 | 필수 | 하네스가 매 요청 앞에 붙이는 지시문. 에이전트의 상시 지침이며 세션 내내 대체로 고정.
- 파일시스템 | Filesystem | 초급 | - | 에이전트가 읽고 쓰고 실행하는 파일·폴더 트리. 코딩 에이전트의 기본 환경.
- 하네스 | Harness | 중급 | 필수 | 모델을 에이전트로 만드는 주변 장치 전부 — 도구·시스템 프롬프트·컨텍스트 관리·권한·훅.
- 환경 | Environment | 중급 | - | 에이전트가 작용하는 세계. 도구 결과로 인식하고 도구 호출로 바꾸는 하네스 바깥 전부.

### 핵심 개념
- AI | AI | 초급 | 필수 | 컴퓨터가 새롭게 잘하게 된 일을 가리키는 움직이는 이름표. 지금은 대규모 언어모델(LLM)을 뜻한다.
- 다음 토큰 예측 | Next-token prediction | 중급 | - | 모델이 실제로 하는 일. 컨텍스트에서 다음 토큰 하나를 뽑아 붙이고 이를 반복한다.
- 모델 | Model | 초급 | 필수 | 학습으로 정해진 수많은 숫자(파라미터) 덩어리. 다음 토큰 예측만 하며 혼자서는 아무것도 못 한다.
- 비결정성 | Non-determinism | 고급 | - | 같은 입력이라도 매번 다른 출력이 나올 수 있는 성질.
- 에이전트 | Agent | 초급 | 필수 | 모델에 도구·시스템 프롬프트·컨텍스트를 붙여 사용자와 주고받으며 일하게 만든 것. '움직이는 모델'.
- 추론 | Inference | 중급 | - | 학습된 모델을 실제로 돌려 출력을 만드는 것. 요청마다 일어나며 파라미터는 고정된다.
- 파라미터(가중치) | Parameters | 중급 | - | 학습으로 조정된 모델 내부의 수십억 개 숫자. 모델이 아는 모든 것이 여기에 담긴다.
- 학습(훈련) | Training | 중급 | - | 방대한 텍스트로 다음 토큰 예측을 잘하도록 파라미터를 조정하는 과정.
